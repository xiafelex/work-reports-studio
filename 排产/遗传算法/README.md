## 使用方法

### 1. 准备数据文件

将以下两个Excel文件放在项目根目录：
- `焊口初始化信息.xlsx`：包含列【管线号、焊口号、加字母焊口号、寸径】
- `施压包划分汇总表.xlsx`：包含工作簿【施压包划分总表】，包含列【管线号、试压包号】

### 2. 运行程序

```bash
python main.py
```

## 完整工作流程

### 阶段1: 数据处理
1. **读取Excel数据**：从两个Excel文件读取原始数据
2. **初始化焊口数据**：
   - 如果焊口号为空，使用加字母焊口号填充
   - 过滤焊口号为空的记录
   - 过滤寸径无效的记录
3. **按管线汇总**：计算每条管线的总寸径和焊口数量
4. **合并施压包信息**：将管线数据与施压包数据合并

### 阶段2: 遗传算法优化
1. **初始化种群**：生成100个初始解（融入领域知识）
2. **迭代优化**：
   - 选择（锦标赛选择）
   - 交叉（顺序交叉OX）
   - 变异（交换/逆序/试压包重排）
   - 精英保留
3. **输出最优解**：最短工期的调度方案

## 算法特点

### 编码方案：单层排列编码（方案1-Simple）
- **染色体**：管线ID的排列，表示焊接优先级
- **解码**：按顺序分配给最早空闲的焊工
- **约束满足**：一条管线由一个焊工从头焊到尾（自动满足"同一时间只有一组"约束）

### 适应度函数
```
适应度 = 1 / (总工期 + 试压包分散惩罚 + 负载不均惩罚)
```

### 遗传算子
- **选择**：锦标赛选择（k=3）
- **交叉**：顺序交叉OX（保证不重复）
- **变异**：交换/逆序/试压包内重排（三种策略随机）

## 配置说明

所有配置项在 `src/config.py` 中，**可根据实际情况灵活修改**：

### 1. 焊工配置（重要，可修改）

```python
WORKER_CONFIG = {
    "teams": {
        1: 60,   # 一队：60组焊工
        2: 25,   # 二队：25组焊工
        3: 25    # 三队：25组焊工
    },
    "daily_capacity": 25  # 一组焊工一天焊接寸数
}
```

**如何修改：**
- **修改焊工数量**：直接改数字，如 `1: 80` 表示一队有80组
- **增加队伍**：添加新行，如 `4: 30` 添加四队30组
- **减少队伍**：删除对应行，如删除 `3: 25` 则只有两个队
- **修改日产能**：修改 `daily_capacity`，如改为 `30` 表示每组每天焊30寸

**示例配置：**
```python
# 只有两个队伍
WORKER_CONFIG = {
    "teams": {
        1: 80,
        2: 30
    },
    "daily_capacity": 30
}

# 五个队伍
WORKER_CONFIG = {
    "teams": {
        1: 40,
        2: 20,
        3: 20,
        4: 15,
        5: 15
    },
    "daily_capacity": 25
}
```

### 2. Excel文件配置

- **文件路径**：可修改Excel文件名和位置
- **工作簿名称**：可修改要读取的工作簿
- **列名映射**：如果Excel列名不同，修改映射关系

### 3. 遗传算法参数

```python
GA_CONFIG = {
    "population_size": 100,    # 种群大小（越大越慢但可能更优）
    "generations": 500,        # 迭代代数（越多越慢但可能更优）
    "crossover_rate": 0.8,     # 交叉率
    "mutation_rate": 0.15,     # 变异率
    "elite_size": 5,           # 精英数量
    "tournament_size": 3       # 锦标赛大小
}
```
