# Work Reports Studio

一个面向汇报材料、PPT 版本演进、分析底稿和项目策划文稿的工作仓库。

这个仓库不是通用知识库，也不是团队全过程归档仓。它的角色是承接“要交付的汇报成果”，并把与汇报直接相关的文稿、版本、引用资料和风格规范组织起来，同时与主记忆仓库 `ai-memory-vault` 配合使用。

## 和主记忆仓库的关系

这个仓库服务于主记忆体，但不替代主记忆体：

- `ai-memory-vault`：沉淀长期记忆、方法论、任务镜像、风格规则
- `work-reports-studio`：维护汇报成品、PPT 版本、文稿底稿、当前项目引用资料

建议协作方式：

1. 在这里完成具体汇报的文稿、PPT、版本迭代和引用资料整编
2. 把最终可复用的结构、风格、判断和任务复盘回写到 `ai-memory-vault`
3. 不把所有原始团队过程资料长期堆在这里

## 两类汇报怎么分

### 1. `01-排产数字化汇报/`

这类内容围绕你的“排产数字化主线”，重点不是单个项目介绍，而是：

- 排产方法
- 进度逻辑
- 数据驱动判断
- 数字化能力如何支撑管理与施工
- 排产相关专题汇报、季度汇报、专题分析

这里当前承接两类东西：

- `文稿与分析/`：直接用于汇报写作的 Markdown 章节、分析稿、阶段基线
- `参考资料/`：进度表、合同执行、图纸清单、物流状态、现场单元信息等支撑资料

同时，下面两处也属于这条主线的重要专题资产：

- [排产](/Users/felex/Documents/Codex/生物柴油模式/排产)
- [排产PPT素材](/Users/felex/Documents/Codex/生物柴油模式/排产PPT素材)

其中：

- `排产/` 更偏专题工作台，包含研究资料和 `遗传算法` 代码资产
- `排产PPT素材/` 更偏复用素材库

### 2. `02-项目介绍汇报/`

这类内容围绕“项目本身怎么介绍、怎么汇报、怎么对外讲清楚”，重点是：

- 项目介绍型 PPT
- Technip 或其他对象的 briefing
- 分工策划与章节组织
- 项目质量、焊接、采购物流等介绍型文稿

这里当前分成：

- `文稿与PPT/`：项目汇报主稿、PPT、不同版本和中英文稿
- `参考资料/`：质量、焊接、格式规范、插图等补充型引用资料

## `90-共享规范/`

这里放跨项目、跨汇报类型都能复用的规则，例如：

- 汇报写作风格
- 项目策划检查清单
- PPT 备注和呈现风格分析
- 内容工程相关总结

这层内容以后如果稳定、长期有效，应该同步沉淀到 `ai-memory-vault` 的 `knowledge` 层。

## 当前项目的理解方式

你现在这个项目，不建议拆成“一个项目一个分支”，而更适合在同一个仓库里保留两个长期稳定的汇报类型：

- 排产数字化主线汇报
- 项目介绍型汇报

也就是说：

- “类型”用目录表达
- “版本迭代”用文件版本和少量分支表达
- “长期方法和风格”回写主记忆仓库

## 分支怎么用

分支不是内容分类工具，而是临时工作线。

### 应该留在 `main` 的内容

- 两类汇报的稳定目录结构
- 关键里程碑版本
- 已经成型的主稿和 PPT
- 重要参考资料
- 稳定可复用的共享规范

### 适合开分支的场景

- 一次大规模改版的汇报重写
- 面向某个特定对象的定制版本
- `排产/遗传算法/` 的实验性调整
- 仓库级目录重构或批量清理

推荐命名：

- `report/technip-briefing-rewrite`
- `report/scheduling-quarterly-deck`
- `experiment/scheduling-ga-constraints`
- `chore/restructure-report-library`

### 不建议用分支代替目录的场景

- “排产数字化”一条分支
- “项目介绍”一条分支
- “马来西亚项目”一条长期常驻分支
- 靠分支来承载所有历史版本

这些都更适合落在目录和文件版本里，而不是长期分支树里。

## 对当前仓库的具体建议

### 建议放在目录里的

- `01-排产数字化汇报/`：排产专题汇报和分析型输出
- `02-项目介绍汇报/`：项目介绍和对外 briefing
- `90-共享规范/`：风格、模板、检查清单
- `排产/遗传算法/`：继续作为排产数字化专题资产维护

### 可以考虑用分支承接的

- 下一轮 Technip 汇报的大改版
- 下一轮排产数字化汇报的新叙事结构
- 遗传算法的实验性约束或指标方案

### 不建议放到分支去“长期保存”的

- 当前这两个汇报大类本身
- 常用参考资料
- 稳定模板与规范

## 推荐工作流

1. 先判断这次工作属于哪一类汇报
2. 把主稿和成品放进对应目录
3. 把引用资料收进该目录下的 `参考资料/`
4. 把跨项目可复用的写法和规则抽到 `90-共享规范/`
5. 任务结束后，把长期有效的结论回写 `ai-memory-vault`

## 当前定位

这个仓库更适合作为：

- 汇报工作台
- 项目介绍与专题分析并行维护的交付仓
- PPT 与文稿版本演进仓
- 连接主记忆仓库的交付侧资产仓
